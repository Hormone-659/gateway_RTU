# core 目录说明

`core` 目录存放与 UI 解耦的领域核心逻辑，供后台服务（`services/`）和 UI（`gateway/`）共用。这些模块不依赖 Tkinter，可以在无头（Headless）环境的工业网关中直接使用。

## 子目录说明

- `alarm/`：报警评估核心逻辑。
  - 负责将各部位的故障等级转换为整体报警等级。
  - 生成 RTU 寄存器映射表。
- `sensor/`：传感器数据处理核心逻辑。
  - 定义传感器数据模型（如振动速度转换）。
  - 提供统一的阈值判断引擎。
- `modbus/`：Modbus 通信封装。
  - 提供统一的 RTU 客户端接口，屏蔽底层库（pymodbus）细节。

## 设计原则

1.  **无 UI 依赖**：所有代码均不应导入 `tkinter` 或 `matplotlib`。
2.  **纯逻辑**：主要处理数据转换、状态评估和协议封装。
3.  **可复用**：同时支持前台 UI 应用和后台 Systemd 服务。
